# CSV Preprocessor Toolkit

Это набор для подготовки тяжелой CSV-выгрузки ОГХ перед подключением к веб-прототипу.

## Зачем нужен
Исходный CSV слишком тяжелый и частично некорректный для прямой загрузки во фронт:
- большой размер;
- смешанные типы данных;
- геометрия и дробные значения ломают обычный CSV-парсинг.

Поэтому сначала надо сделать подготовленный слой данных.

## Что делает скрипт
`preprocess_csv.py`:
1. читает исходный CSV;
2. сохраняет полный список заголовков;
3. делит колонки на 3 логических набора:
   - Объекты ОДХ
   - Работы на ОДХ
   - Работы на ОКБ
4. формирует `frontend_sample.json` для подключения в прототип;
5. формирует отчет `parse_report.json`.

## Что получится на выходе
В папке `prepared_output` появятся:
- `all_headers.json`
- `datasets_meta.json`
- `frontend_sample.json`
- `frontend_sample.js`
- `warnings_examples.json`
- `parse_report.json`

## Как запускать
Положи исходный CSV рядом со скриптом и запусти:

```bash
python preprocess_csv.py
```

## Важное ограничение
Это эвристический препроцессор.

Он подходит для:
- прототипа,
- анализа структуры,
- подготовки выборки для интерфейса.

Для production лучше попросить:
- TSV,
- XLSX,
- или корректно сформированный CSV с нормальным разделителем и кавычками.
