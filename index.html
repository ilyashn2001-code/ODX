МосДата · Благоустройство · Витрина сценариев

Предиктивный анализ объектов дорожного хозяйства

Кликабельный прототип для отбора и приоритизации ОДХ по критериям покрытия, объема, стоимости и давности ремонта. Пользователь задает критерии, настраивает веса и получает реестр объектов вместе с картой проблемных зон.

Критерии отбора

Минимум один критерий обязателен. Можно использовать системные и пользовательские правила с группировками.

Конструктор системного критерия Шаг 1
Примеры: процент покрытия 0–85, рекомендуемый объем 3000–999999, давность ремонта 5–20.
Конструктор пользовательского критерия Шаг 2
Пользовательский критерий добавляется как готовая логическая группа. Для прототипа доступны шаблоны с И/ИЛИ.

Приоритизация

Если веса не заданы, применяется системная модель. Пользовательские веса влияют только на сортировку списка.

Конструктор системного веса Шаг 3
Сумма пользовательских весов должна составлять 100%. Иначе будет применена системная модель.

Карта

Подложка OpenStreetMap. На карте отображаются проблемные территории и цветовая дифференциация по приоритету.
